# 常见失败原因

本页综合 15 个钻具断落案例页与标准页，整理处置失败或周期拉长的常见原因。它用于提醒 Agent 和现场工程师补齐信息、设置判断节点，不替代原始案例复核。

## 失败原因总览

| 类型 | 表现 | 后果 | 预防措施 |
| --- | --- | --- | --- |
| 信息不足 | 扣型、鱼顶、井斜、落鱼长度不清 | 工具选型不确定，容易误下工具 | 先补充关键字段，必要时通井、打印、探鱼顶 |
| 井眼未清洁 | 砂埋、返砂异常、鱼顶覆盖 | 打捞工具无法有效抓取或造扣 | 先循环洗井/冲砂，确认返砂变化 |
| 强卡未解除 | 上提无效、活动无改善 | 直接打捞成功率低，可能二次卡钻 | 先判断卡因，按解卡路径逐级处理 |
| 工具不匹配 | 内捞/外捞尺寸、扣型、外径不匹配 | 抓取失败、工具损坏或落井 | 按 [[打捞工具目录]] 复核规格和适用范围 |
| 参数照搬 | 直接套用历史案例拉力、扭矩、震击参数 | 超载、断脱或事故扩大 | 参数必须经管柱强度和现场设计确认 |
| 水平井特殊性低估 | 摩阻大、携砂差、工具通过性不足 | 冲砂失败、卡阻加重、工具不到位 | 按 [[水平井特殊工艺]] 单独评估 |
| 升级节点不清 | 多次重复无效打捞 | 周期拉长，井况变差 | 预设停止条件、升级条件和侧钻/弃鱼论证点 |

## 案例复盘关注点

阅读案例页时，优先抽取以下内容：

1. 事故原因：疲劳、刺漏、扣型问题、卡钻、井眼复杂或操作因素。
2. 鱼顶状态：清楚、变形、砂埋、遇阻、强卡。
3. 首次处置：是否先循环、解卡、探鱼顶或直接打捞。
4. 工具链路：捞矛、打捞筒、公锥、母锥、震击器、套铣、磨铣等。
5. 失败转折：何时认定无效，何时升级。
6. 最终结果：成功起出、部分起出、侧钻、弃鱼或其他。

## 针对当前系统的提示词约束

Agent 输出中如果出现以下情况，必须降级为“待确认”或“工程推断”：

- 用户未提供扣型，却给出确定转换接头或造扣工具型号。
- 用户未提供井斜轨迹，却对水平井摩阻、携砂能力下确定结论。
- 用户未提供钻具强度，却给出精确上提力、震击力、扭矩或钻压。
- 用户只描述“上提未动”，却直接跳过循环洗井、鱼顶确认和卡因判断。
- 没有引用标准页或案例页，却把建议写成强制结论。

## 失败到升级的判断节点

| 阶段 | 失败信号 | 下一步 |
| --- | --- | --- |
| 井况复核 | 信息无法闭合或井控异常 | 暂停处置，补充资料和现场设计 |
| 循环洗井/冲砂 | 返砂持续异常、鱼顶仍被覆盖 | 继续清洁或考虑套铣前处理 |
| 活动/浸泡/震击 | 悬重、扭矩无改善或风险升高 | 停止重复操作，复核卡因 |
| 常规打捞 | 多次抓取失败、鱼顶损伤 | 修鱼顶、换工具或套铣/磨铣 |
| 套铣/磨铣 | 扭矩异常、循环异常、工具风险升高 | 停止升级，论证侧钻或弃鱼 |
